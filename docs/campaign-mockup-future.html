Type selector changes the form: "Completions" shows quest/saga picker + count. "Level" shows skill/attribute picker + level. "XP earned" shows attribute picker + XP amount. All types can optionally add "within N days."
Impact on initial implementation:
The criterion's target_type field should be designed as an extensible string enum, not a boolean quest/saga flag. Initial values: "quest_completions", "saga_completions".
Future values: "skill_level", "attr_level", "attr_xp", "skill_xp".
The target_count field generalizes naturally (completions count, level target, XP target).
Time-bounded criteria add an optional time_window_days field (null = no window).
